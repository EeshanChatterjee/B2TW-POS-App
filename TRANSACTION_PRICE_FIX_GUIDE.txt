PROBLEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Order items, orders, and bills may have prices that don't match current menu
prices. This could happen due to:
  • Menu price changes after orders were placed
  • Price calculation errors
  • Data migration issues
  • Manual database modifications

SOLUTION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Two scripts have been created:

  1. validate-transaction-prices.js   → Analyze WITHOUT making changes
  2. fix-transaction-prices.js        → Fix all discrepancies

WHAT GETS FIXED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

order_items Table:
  • unit_price → Updated to match current menu price
  • total_price → Recalculated as quantity × unit_price

orders Table:
  • total_amount → Recalculated from SUM of order_items.total_price
  • updated_at → Set to current timestamp

bills Table:
  • total_amount → Updated to match their corresponding order.total_amount
  • updated_at → Set to current timestamp

EXAMPLES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Example 1: Item with old price
  Product "Bao" current price: ₹160
  Order item stored price: ₹150
  → Fix will update to ₹160

Example 2: Order with incorrect total
  Order total stored: ₹450.50
  Correct total (sum of items): ₹480.00
  → Fix will recalculate to ₹480.00

Example 3: Bill not matching order
  Bill amount: ₹450.50
  Order amount: ₹480.00
  → Fix will update bill to ₹480.00
